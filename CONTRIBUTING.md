# How to contribute to SpaceWasm

## Bug Reports

If you are working with SpaceWasm or find some issue with the documentation, interpreter, tests, or executables,
please file a bug report in [GitHub](https://github.com/nasa/spacewasm/issues).

## Bug Fixes

If you are interested in contributing code to SpaceWasm, bugs already
reported in [GitHub Issues](https://github.com/nasa/spacewasm/issues)
labeled with [good first issue](https://github.com/nasa/spacewasm/issues?q=is%3Aissue%20state%3Aopen%20label%3A%22good%20first%20issue%22) are a good first start.

## Bigger Pieces of Work

We generally will heavily gate the development of major features and Wasm
proposals. If you would like to contribute a larger feature such as a post-MVP
WebAssembly proposal, please create a new issue labeled with
[WebAssembly Proposal](https://github.com/nasa/spacewasm/issues?q=is%3Aissue%20state%3Aopen%20label%3A%22WebAssembly%20Proposal%22).
We will work with you to determine if the work is feasible to merge
from external contributors.

Contributing large features may require significant coordination with the
maintainers, and we ask that you open a proposal issue before investing
substantial effort so we can align on scope and design up front.

Please also understand that SpaceWasm has a specific set of goals, quality
standards, and long-term maintenance considerations. As maintainers, we reserve
the discretion to decline or close any contribution that we feel does not align
with the project's direction, even if the work itself is sound. We will always
try to explain our reasoning and, wherever possible, suggest a path forward. We
greatly appreciate every contribution, and we hope opening a discussion early
helps avoid disappointment on work that may not be a good fit.
